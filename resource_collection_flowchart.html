Resource Collection Flowchart

How the /ask endpoint collects system resource metrics

POST /ask Request Received
Phase 1 — Timing
Start Timer
start_time = time.time()
Send Prompt to Ollama
POST {OLLAMA_BASE_URL}/api/generate
Calculate Total Duration
total_duration = time.time() - start_time
Phase 2 — Ollama Metrics (from response)
Extract Ollama Timing Data
load_duration → model load time (ns → sec)
prompt_eval_duration → prompt processing time
eval_duration → response generation time
Extract Token Counts
prompt_eval_count → prompt tokens
eval_count → response tokens
total_tokens = prompt + response
Calculate Tokens/Second
response_tokens / (eval_duration / 1B)
Phase 3 — System & GPU Resources (parallel)
_get_system_resources()
Uses psutil library
Read CPU Usage
psutil.cpu_percent(interval=0.1)
Read Memory Usage
psutil.virtual_memory()
→ used_mb, total_mb, percent
Return System Metrics
cpu_usage_percent
memory_used_mb / total_mb / percent
On Exception → Return {}
Fields will be null
_get_gpu_info()
Uses nvidia-smi CLI
Run nvidia-smi
nvidia-smi --query-gpu=
name,utilization.gpu,
memory.used,memory.total
nvidia-smi OK?
YES → Parse GPU Data
gpu_name, gpu_usage_%
gpu_mem_used / total / %
NO → Return {}
No GPU / not found / timeout
Fields will be null
Phase 4 — Combine & Return
Build ResourceUsage Object
Merge: Ollama timing + Token counts + System metrics + GPU info
Return AskResponse
{ model, question, answer, structured, resource_usage }
Response Sent to Client
